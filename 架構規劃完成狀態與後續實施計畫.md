### Sprint B（1-2週）：Windows Cloud API 整合

#### B1. Windows CloudAPIService 實作
```csharp
// Windows/Core/Services/CloudAPIService.cs（新建）
public class CloudAPIService : ICloudAPIService
{
    private readonly HttpClient _client;
    private readonly string _baseUrl;

    public async Task<WoundAnalysisResult> AnalyzeAsync(
        byte[] imageData, CalibrationData calibration)
    {
        using var form = new MultipartFormDataContent();
        form.Add(new ByteArrayContent(imageData), "image", "wound.jpg");
        form.Add(new StringContent(JsonSerializer.Serialize(calibration)),
                 "calibration");

        var response = await _client.PostAsync("/api/v1/analyze", form);
        response.EnsureSuccessStatusCode();

        var json = await response.Content.ReadAsStringAsync();
        return JsonSerializer.Deserialize<WoundAnalysisResult>(json);
    }
}
```

```kotlin
// 使用 ML Depth Estimation 替代平面假設
class DepthEstimationModule(context: Context) {
    // 選項A：Google ARCore Depth API（支援裝置）
    // 選項B：MiDaS 深度估算 TFLite 模型
    // 選項C：單目深度估算（相對距離）

    fun estimateDepth(bitmap: Bitmap): DepthMap {
        return when {
            isARCoreDepthSupported() -> estimateViaARCore(bitmap)
            else -> estimateViaMiDaS(bitmap)  // fallback
        }
    }
}
```

#### C3. 分類集成（Android）
```kotlin
// 加入多模型投票機制（對應 iOS MultiAlgorithmDetector）
class MultiModelClassifier(context: Context) {
    private val models = listOf(
        WoundTypeClassifier(context, "wound_type_v2.tflite"),
        SeverityClassifier(context, "severity_v1.tflite"),
        TissueClassifier(context, "tissue_v1.tflite")
    )

    fun classify(bitmap: Bitmap, mask: Bitmap): EnsembleResult {
        val results = models.map { it.classify(bitmap, mask) }
        return EnsembleResult(
            woundType = voteMajority(results.map { it.woundType }),
            confidence = results.map { it.confidence }.average().toFloat(),
            tissueComposition = mergeTissueResults(results)
        )
    }
}
```

## 5. Phase 2 臨床整合規劃（新增）

### 5.1 FHIR R4 整合架構
```
WoundAI ←→ FHIR Server ←→ EMR/EHR

資源對應：
- Patient → PatientEntity（患者資料）
- Observation → WoundRecord（量測記錄）
- DiagnosticReport → PDFReport（診斷報告）
- Media → WoundImage（傷口影像）
- Practitioner → DoctorProfile（醫師資料）
```

```python
# 雲端 AI 服務 / FHIR 客戶端
from fhir.resources.observation import Observation
from fhir.resources.patient import Patient

class FHIRExportService:
    def wound_record_to_observation(self, record: WoundRecord) -> Observation:
        return Observation(
            status="final",
            code={"coding": [{"system": "http://loinc.org",
                              "code": "72298-5",  # Wound area
                              "display": "Wound area"}]},
            subject={"reference": f"Patient/{record.patient_id}"},
            valueQuantity={
                "value": record.wound_area_cm2,
                "unit": "cm2",
                "system": "http://unitsofmeasure.org"
            },
            component=[
                {"code": {"text": "wound_type"},
                 "valueString": record.wound_type},
                {"code": {"text": "severity"},
                 "valueInteger": record.severity_score}
            ]
        )
```

### 5.2 臨床決策支援（CDS Hooks）
```
觸發點：
- 每次量測完成 → 推薦敷料更換時間
- 連續惡化趨勢 → 警示升級護理等級
- 癒合停滯 > 2週 → 推薦轉介專科

規則引擎（簡化）：
IF wound_area 增加 > 20% over 7 days
  THEN alert("傷口惡化，建議醫師評估")

IF tissue_composition.necrotic > 40%
  THEN recommend("考慮清創處置")
```
