# 規則式嚴重度分數卡與治療建議 — 臨床參數表（Phase 1 #2）

> **可解釋、臨床可調、無黑箱**。所有輸出皆為**輔助用途、需醫師確認**（`status: rule_based`）。
> 參數定義於 `rules_params.json`，臨床顧問可調整後以 `load_params()` 載入；本表為審閱與校準依據。

## 1. 參數定義

| 參數 | 預設值 | 意義 | 邊界語意 |
|---|---|---|---|
| `area_thresholds_cm2` | `[4, 16]` | 傷口面積（cm²）分級門檻 | 半開區間：`area < 4` → 0 分；`4 ≤ area < 16` → 1 分；`area ≥ 16` → 2 分 |
| `necrosis_frac` | `0.2` | 壞死組織占比門檻 | **嚴格大於**：`necrosis > 0.2` 才計分（+2） |
| `slough_frac` | `0.3` | 腐肉組織占比門檻 | **嚴格大於**：`slough > 0.3` 才計分（+1，且僅在未計壞死分時） |
| `grade_base` | `1` | 等級基底 | 最終 grade = `1 + min(points, 3)` |
| `grade_max` | `4` | 等級上限 | points 上限 4，但 grade 截斷於 4 |

## 2. 分數卡邏輯（`severity_scorecard`）

```
面積分 = 0 (area<4) | 1 (4≤area<16) | 2 (area≥16)
組織分 = 2 (necrosis>0.2)  否則  1 (slough>0.3)  否則  0      ← 壞死優先於腐肉
points = 面積分 + 組織分        （理論最大 4）
grade  = 1 + min(points, 3)     （落在 1–4）
```

| grade | points | 典型情境 |
|---|---|---|
| 1 | 0 | 小面積、肉芽為主 |
| 2 | 1 | 中面積，或輕度腐肉 |
| 3 | 2 | 大面積，或腐肉偏多 |
| 4 | 3–4 | 大面積且壞死，最高警示 |

## 3. 治療方向建議（`treatment`，4 條互斥路徑，依序判斷）

| 優先序 | 條件 | 建議 |
|---|---|---|
| 1 | `necrosis > necrosis_frac` | 疑壞死組織 → 建議清創評估／轉診 |
| 2 | `slough > slough_frac` | 腐肉偏多 → 清創與適當敷料 |
| 3 | `grade ≥ 3`（無壞死／腐肉） | 情況偏重 → 轉介傷口照護專科評估 |
| 4 | 其餘 | 肉芽為主 → 維持濕潤敷料、定期追蹤 |

## 4. 驗證

`test_clinical_rules.py`（CI 強制）涵蓋 **30 項**：面積/組織門檻邊界（含半開區間與嚴格大於語意）、壞死優先序、grade 上限與範圍、單調性、自訂參數覆寫、缺鍵容錯、決定性、4 條治療分支、參數檔 round-trip。**30/30 PASS**。

門檻為**初始預設**，非臨床定論。建議於試點期以醫師標註對照逐步校準 `area_thresholds_cm2` 與 `necrosis_frac/slough_frac`，並記錄版本。規則式設計確保每個 grade 與建議皆可回溯到明確條件（見每筆輸出之 `explain` 欄位），符合可解釋 AI 與醫療決策可審計需求。
