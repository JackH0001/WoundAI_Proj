## 資料庫設計

### 4.1 關聯式資料庫 (PostgreSQL)

#### 核心資料表結構

##### 4.1.1 患者資料表
```sql
CREATE TABLE patients (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    patient_code VARCHAR(50) UNIQUE NOT NULL, -- 匿名化患者代碼
    age_range VARCHAR(20), -- "18-25", "26-35" 等
    gender VARCHAR(10),
    comorbidities JSONB,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    -- 隱私保護：不存儲真實姓名或身份證明
    CONSTRAINT check_age_range CHECK (age_range IN ('0-17', '18-25', '26-35', '36-45', '46-55', '56-65', '66-75', '75+'))
);

CREATE INDEX idx_patients_patient_code ON patients(patient_code);
CREATE INDEX idx_patients_created_at ON patients(created_at);
```

##### 4.1.2 傷口記錄表
```sql
CREATE TABLE wound_records (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    patient_id UUID REFERENCES patients(id) ON DELETE CASCADE,
    
    -- 影像資料
    image_url VARCHAR(500) NOT NULL,
    thumbnail_url VARCHAR(500),
    depth_data_url VARCHAR(500),
    
    -- 測量結果
    area_cm2 DECIMAL(10,4),
    volume_cm3 DECIMAL(10,4),
    max_depth_mm DECIMAL(8,2),
    avg_depth_mm DECIMAL(8,2),
    perimeter_cm DECIMAL(10,2),
    
    -- 分類結果
    classification JSONB NOT NULL,
    tissue_analysis JSONB,
    quality_metrics JSONB,
    
    -- 元資料
    capture_metadata JSONB,
    device_info JSONB,
    
    -- 時間戳記
    captured_at TIMESTAMP WITH TIME ZONE NOT NULL,
    analyzed_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    -- 品質控制
    validation_status VARCHAR(20) DEFAULT 'pending',
    validator_id UUID,
    validation_notes TEXT,
    
    CONSTRAINT check_validation_status CHECK (validation_status IN ('pending', 'approved', 'rejected', 'needs_review'))
);

-- 索引優化
CREATE INDEX idx_wound_records_patient_id ON wound_records(patient_id);
CREATE INDEX idx_wound_records_captured_at ON wound_records(captured_at);
CREATE INDEX idx_wound_records_validation_status ON wound_records(validation_status);
CREATE INDEX idx_wound_records_classification ON wound_records USING GIN (classification);
```

##### 4.1.3 AI模型版本表
```sql
CREATE TABLE ai_model_versions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    model_name VARCHAR(100) NOT NULL,
    version VARCHAR(20) NOT NULL,
    model_type VARCHAR(50) NOT NULL, -- 'classification', 'segmentation', 'measurement'
    
    -- 模型資訊
    model_url VARCHAR(500) NOT NULL,
    model_size_bytes BIGINT,
    input_shape JSONB,
    output_shape JSONB,
    
    -- 效能指標
    accuracy DECIMAL(5,4),
    precision_score DECIMAL(5,4),
    recall DECIMAL(5,4),
    f1_score DECIMAL(5,4),
    
    -- 版本管理
    is_active BOOLEAN DEFAULT FALSE,
    deployment_date TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    UNIQUE(model_name, version)
);

CREATE INDEX idx_ai_models_active ON ai_model_versions(model_name, is_active) WHERE is_active = TRUE;
```

##### 4.1.4 分析任務表
```sql
CREATE TABLE analysis_tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    wound_record_id UUID REFERENCES wound_records(id) ON DELETE CASCADE,
    
    -- 任務狀態
    status VARCHAR(20) DEFAULT 'pending',
    priority INTEGER DEFAULT 5, -- 1-10, 1為最高優先級
    
    -- 處理資訊
    assigned_worker VARCHAR(100),
    started_at TIMESTAMP WITH TIME ZONE,
    completed_at TIMESTAMP WITH TIME ZONE,
    processing_time_ms INTEGER,
    
    -- 錯誤處理
    error_message TEXT,
    retry_count INTEGER DEFAULT 0,
    max_retries INTEGER DEFAULT 3,
    
    -- 資源使用
    cpu_usage DECIMAL(5,2),
    memory_usage_mb INTEGER,
    gpu_usage DECIMAL(5,2),
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    CONSTRAINT check_status CHECK (status IN ('pending', 'processing', 'completed', 'failed', 'cancelled')),
    CONSTRAINT check_priority CHECK (priority BETWEEN 1 AND 10)
);

CREATE INDEX idx_analysis_tasks_status ON analysis_tasks(status);
CREATE INDEX idx_analysis_tasks_priority ON analysis_tasks(priority DESC);
CREATE INDEX idx_analysis_tasks_created_at ON analysis_tasks(created_at);
```

## AI模型規格

### 5.1 傷口分割模型

#### 5.1.1 U-Net架構規格
```yaml
模型名稱: WoundSegmentationUNet_v2.1
架構: U-Net with ResNet50 backbone
輸入規格:
  - 影像尺寸: 512x512x3 (RGB)
  - 數據類型: float32
  - 值範圍: [0, 1]
  - 預處理: 標準化 (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])

輸出規格:
  - 分割遮罩: 512x512x1 (單通道)
  - 數據類型: float32  
  - 值範圍: [0, 1] (機率值)
  - 後處理: 閾值化 (threshold=0.5)

效能指標:
  - IoU (Intersection over Union): 0.89
  - Dice Coefficient: 0.93
  - Pixel Accuracy: 0.96
  - 推論時間: <200ms (iPhone 15 Pro)
  - 模型大小: 25.6 MB

訓練資料:
  - 訓練集: 15,000張標註影像
  - 驗證集: 3,000張標註影像
  - 測試集: 2,000張標註影像
  - 資料來源: 多中心醫院 + 公開資料集
```

#### 5.1.2 模型轉換格式
```python
# TensorFlow to CoreML conversion
import coremltools as ct
import tensorflow as tf

# 載入TensorFlow模型
model = tf.keras.models.load_model('wound_segmentation_unet_v2.1.h5')

# 轉換為CoreML格式
coreml_model = ct.convert(
    model,
    inputs=[ct.ImageType(
        name="input_image",
        shape=(1, 512, 512, 3),
        scale=1/255.0,
        bias=[0, 0, 0]
    )]
)

# 設定元資料
coreml_model.short_description = "Wound segmentation model"
coreml_model.version = "2.1"
coreml_model.author = "WoundAI Team"

# 儲存CoreML模型
coreml_model.save('WoundSegmentation.mlmodel')
```

### 5.3 組織分析模型

#### 5.3.1 語義分割架構
```yaml
模型名稱: TissueSegmentation_v1.2
架構: DeepLabV3+ with Xception backbone
輸入規格:
  - 影像尺寸: 512x512x3
  - 數據類型: float32

輸出規格:
  - 類別數: 5 (健康皮膚, 肉芽組織, 纖維組織, 壞死組織, 痂皮)
  - 輸出尺寸: 512x512x5
  - 激活函數: Softmax

類別定義:
  - Class 0: 健康皮膚 (Healthy Skin)
  - Class 1: 肉芽組織 (Granulation Tissue)  
  - Class 2: 纖維組織 (Fibrous Tissue)
  - Class 3: 壞死組織 (Necrotic Tissue)
  - Class 4: 痂皮 (Eschar)
```

```python
import hashlib
import uuid
from datetime import datetime

class DataAnonymizer:
    def __init__(self, salt: str):
        self.salt = salt
    
    def anonymize_patient_id(self, patient_id: str) -> str:
        """生成匿名化患者ID"""
        hash_input = f"{patient_id}{self.salt}"
        return hashlib.sha256(hash_input.encode()).hexdigest()[:16]
    
    def anonymize_image_metadata(self, metadata: dict) -> dict:
        """移除影像中的敏感元資料"""
        sensitive_fields = [
            'gps_latitude', 'gps_longitude', 
            'device_serial', 'user_comment',
            'camera_serial_number'
        ]
        
        anonymized = metadata.copy()
        for field in sensitive_fields:
            anonymized.pop(field, None)
            
        return anonymized
    
    def generate_pseudonym(self, real_id: str) -> str:
        """生成假名ID用於研究"""
        namespace = uuid.UUID('6ba7b810-9dad-11d1-80b4-00c04fd430c8')
        return str(uuid.uuid5(namespace, f"{real_id}{self.salt}"))
```

```python
        })

        status_response = api_client.get(f'/analysis/{analysis_id}/status')
        assert status_response.status_code == 200
        
        # 3. 等待分析完成
        result_response = self._wait_for_completion(api_client, analysis_id)
        assert result_response.status_code == 200
        
        # 4. 驗證結果格式
        result = result_response.json()
        assert 'wound_classification' in result
        assert 'measurements' in result
        assert result['wound_classification']['confidence'] > 0.5
    
    def _wait_for_completion(self, client, analysis_id, timeout=30):
        import time
        start_time = time.time()
        
        while time.time() - start_time < timeout:
            response = client.get(f'/analysis/{analysis_id}')
            if response.json().get('status') == 'completed':
                return response
            time.sleep(1)
        
        raise TimeoutError("Analysis did not complete within timeout")
```

```yaml
    labels:
```

```yaml
      summary: "AI service is down"

  - alert: DatabaseConnectionPoolHigh

    for: 1m
    labels:

    annotations:

  # 記憶體使用率過高
```

### 10.4 備份與災難恢復

#### 10.4.1 資料備份策略
```bash
#!/bin/bash
# backup_script.sh

# 資料庫備份
DB_BACKUP_DIR="/backups/database"
DATE=$(date +%Y%m%d_%H%M%S)

# PostgreSQL完整備份
pg_dump -h postgres-server -U woundai_user -d woundai | gzip > \
  "$DB_BACKUP_DIR/woundai_full_$DATE.sql.gz"

# 增量備份 (WAL)
pg_basebackup -h postgres-server -U replication_user -D \
  "$DB_BACKUP_DIR/incremental_$DATE" -Ft -z -P

# 影像檔案備份到S3
aws s3 sync /app/uploads s3://woundai-backups/images/ \
  --exclude "*.tmp" --delete

# Redis資料備份
redis-cli --rdb "$DB_BACKUP_DIR/redis_$DATE.rdb"

# 清理舊備份（保留30天）
find $DB_BACKUP_DIR -name "*.gz" -mtime +30 -delete

# 備份驗證
if [ $? -eq 0 ]; then
    echo "Backup completed successfully at $DATE" | \
    mail -s "WoundAI Backup Success" admin@woundai.com
else
    echo "Backup failed at $DATE" | \
    mail -s "WoundAI Backup FAILED" admin@woundai.com
fi
```
