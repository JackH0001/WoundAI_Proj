# 醫師標註功能整合說明

## 📋 功能概述

已成功將醫師專用的傷口標註功能整合到 Android 應用程式中，支援 BJWAT 和 revPWAT 評估標準，並可將標註資料上傳到雲端 AI 訓練資料庫。

## 🚀 新增功能

### 1. 醫師身份驗證 (DoctorAuthActivity)
- **功能**: 驗證合格註冊醫師身份
- **驗證方式**: 醫師執照號碼 + 密碼 + 醫院
- **安全機制**: JWT Token 認證
- **權限控制**: 僅限已認證醫師使用

### 2. 標註界面 (AnnotationActivity)
- **影像選擇**: 支援從相簿選擇傷口影像
- **BJWAT 評估**: 11項評估標準的完整標註
- **revPWAT 評估**: 6項評估標準的完整標註
- **即時預覽**: 標註結果即時顯示

### 3. 資料模型 (WoundAnnotationData)
- **完整資料結構**: 包含所有評估項目
- **自動計算**: BJWAT 和 revPWAT 總分
- **嚴重程度評估**: 自動判斷傷口嚴重程度
- **JSON 格式**: 支援雲端傳輸

### 4. 雲端連接 (CloudServiceConnector)
- **資料上傳**: 安全的 HTTPS 上傳
- **離線支援**: 網路中斷時本地儲存
- **狀態追蹤**: 上傳進度和狀態查詢
- **錯誤處理**: 完整的錯誤處理機制

## 📱 使用流程

### 1. 醫師登入
```
主選單 → 醫師標註系統 → 身份驗證
```

**測試帳號**:
- 醫師ID: `D001`, 密碼: `REMOVED_USE_BACKEND_AUTH`
- 醫師ID: `D002`, 密碼: `REMOVED_USE_BACKEND_AUTH`
- 醫師ID: `D003`, 密碼: `REMOVED_USE_BACKEND_AUTH`

### 2. 標註流程
```
選擇影像 → BJWAT評估 → revPWAT評估 → 儲存標註
```

### 3. 資料上傳
```
本地儲存 → 雲端上傳 → 品質控制 → AI訓練
```

## 📊 評估標準

### BJWAT 評估 (11項)
1. **傷口大小 (Size)**: 0-5 級
2. **傷口深度 (Depth)**: 0-4 級
3. **邊緣 (Edges)**: 0-2 級
4. **壞死組織類型 (Necrotic Type)**: 0-3 級
5. **壞死量 (Necrotic Amount)**: 0-4 級
6. **分泌物量 (Exudate Amount)**: 0-4 級
7. **分泌物類型 (Exudate Type)**: 0-3 級
8. **組織顏色 (Tissue Color)**: 0-3 級
9. **肉芽 (Granulation)**: 0-3 級
10. **再上皮化 (Epithelialization)**: 0-3 級
11. **周邊皮膚 (Peri-skin)**: 0-3 級

### revPWAT 評估 (6項)
1. **壞死組織 (Necrosis)**: 1-3 級
2. **Slough (腐肉)**: 1-3 級
3. **肉芽組織 (Granulation)**: 1-3 級
4. **分泌物 (Exudate)**: 1-4 級
5. **顏色 (Color)**: 0-2 級
6. **深度 (Depth)**: 1-3 級

## 🔍 故障排除

### 常見問題

#### 1. 醫師登入失敗
```
問題: 身份驗證失敗
解決: 檢查醫師ID和密碼是否正確
```

#### 2. 影像載入失敗
```
問題: 無法選擇或載入影像
解決: 檢查相機權限和儲存權限
```

#### 3. 上傳失敗
```
問題: 資料上傳失敗
解決: 檢查網路連線和雲端服務狀態
```

#### 4. 應用程式崩潰
```
問題: 標註過程中應用程式崩潰
解決: 檢查記憶體使用和影像大小
```

### 除錯方法
1. 查看 Logcat 日誌
2. 檢查網路連線
3. 驗證權限設定
4. 測試離線功能
